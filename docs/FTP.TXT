FTP.EXE - FTP control-channel smoke test
========================================

Purpose
-------
  Opens an FTP control connection through SprinterESP in ESP-AT
  single-connection TCP mode, verifies the login path and parses the passive
  endpoint returned by PASV. This is the first FTP development step; passive
  data transfers are planned next.

Usage
-----
  FTP.EXE host[:port] [user [password]]

Run NETUP first. FTP.EXE does not join Wi-Fi by itself; it only opens the
FTP control connection after the shared ESP/UART configuration is ready.

Examples
--------
  FTP.EXE 192.168.1.36
  FTP.EXE 192.168.1.36:2121 test secret
  FTP.EXE ftp.example.org anonymous sprinter@example

What It Does
------------
  1. Initializes the Sprinter-WiFi UART using NET.CFG.
  2. Applies the configured ESP UART speed and RTS/CTS flow control.
  3. Keeps ESP-AT in single-connection mode with AT+CIPMUX=0.
  4. Opens a TCP control connection to the FTP server.
  5. Reads and prints FTP reply lines.
  6. Sends USER, PASS if needed, TYPE I, PASV and QUIT.
  7. Parses the 227 passive endpoint and prints host:port.

Current Limits
--------------
  - PASV is parsed, but no RETR/STOR/LIST data transfer yet;
  - no ftp:// URL parser yet;
  - password is passed on the command line for this early diagnostic build;
  - use a simple test FTP server first.
  - passive data transfers will require a separate data connection and may
    enable ESP-AT multi-connection mode only in that later code path.

If startup fails with ESP communication error #1/#4 before "Connecting to",
the failure is in ESP-AT/UART initialization, not in the FTP server. Run
NETRESET, NETUP, then retry FTP.EXE.

Exit Codes
----------
  0  FTP control-channel login smoke test completed.
  1  Invalid command line.
  2  Sprinter-WiFi card was not found.
  3  ESP/TCP communication error.
  4  FTP server returned an unexpected or negative reply.
