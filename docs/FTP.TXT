FTP.EXE - FTP passive file download and directory listing
=========================================================

Purpose
-------
  Opens an FTP control connection through SprinterESP, logs in, enters passive
  mode and downloads a remote file with RETR. Directory listing is available
  explicitly with LIST or NLST flags.

Usage
-----
  FTP.EXE host[:port] file [-o output] [-u user] [-p pass] [-y]
  FTP.EXE host[:port] [path] -l [-u user] [-p pass]
  FTP.EXE host[:port] [path] -n [-u user] [-p pass]
  FTP.EXE /?

Run NETUP first. FTP.EXE does not join Wi-Fi by itself; it only opens the
FTP control connection after the shared ESP/UART configuration is ready.

Examples
--------
  FTP.EXE 192.168.1.36 fbird.zip
  FTP.EXE 192.168.1.36 fbird.zip -o folder\newfile.zip
  FTP.EXE 192.168.1.36 fbird.zip -y
  FTP.EXE 192.168.1.36 -l
  FTP.EXE 192.168.1.36 pub -n
  FTP.EXE 192.168.1.36:2121 -l -u test -p secret
  FTP.EXE ftp.example.org / -l -u anonymous -p anonymous@

What It Does
------------
  1. Initializes the Sprinter-WiFi UART using NET.CFG.
  2. Applies the configured ESP UART speed and RTS/CTS flow control.
  3. Uses ESP-AT multi-connection mode with AT+CIPMUX=1.
  4. Opens the FTP control TCP connection as link 0.
  5. Reads and prints FTP reply lines.
  6. Sends USER, PASS if needed, TYPE I and PASV.
  7. Parses the 227 passive endpoint and prints host:port.
  8. Opens the passive data TCP connection as link 1.
  9. Sends RETR file, LIST path, or NLST path.
  10. Writes downloaded file data to disk or prints directory listing.
      If the local output exists, asks before overwriting. Use -y to overwrite
      without prompting.
  11. Sends QUIT and closes the control connection.

Current Limits
--------------
  - file download is implemented with RETR;
  - directory listing modes are implemented with explicit -l and -n flags;
  - STOR upload is not implemented yet;
  - FTP.EXE uses the same flag style as the RTL8019AS network kit for the
    implemented GET/listing modes. PUT upload is reserved until STOR is
    implemented in this SprinterESP build;
  - password is passed on the command line;
  - use a simple test FTP server first.

If startup fails with ESP communication error #1/#4 before "Connecting to",
the failure is in ESP-AT/UART initialization, not in the FTP server. Run
NETRESET, NETUP, then retry FTP.EXE.

Exit Codes
----------
  0  FTP login, PASV and file download or directory listing completed.
  1  Invalid command line.
  2  Sprinter-WiFi card was not found.
  3  ESP/TCP communication error.
  4  FTP server returned an unexpected or negative reply.
  5  Local file create, write, or close failed.
