NETUP.EXE - Bring the network up

Purpose
  Applies NET.CFG settings and connects SprinterESP / Sprinter-WiFi to Wi-Fi.

Usage
  NETUP.EXE

Before running
  Run NETCFG.EXE /W first and save a valid NET.CFG with SSID and PASS.

Typical sequence
  NETCFG.EXE /W
  NETUP.EXE
  PING.EXE example.com

Behavior
  NETUP loads NET.CFG from the directory that contains NETUP.EXE. This also
  works when NETUP.EXE is started through PATH while another directory is
  current. Keep the runtime configuration beside NETUP.EXE.

  DHCP is enabled when DHCP=1. Static IP, gateway and netmask are applied when
  DHCP=0.

  BAUD controls the UART speed used by automated tools. Supported values are
  230400, 115200, 57600, 38400, 19200 and 9600. Use 57600 or 38400 if 115200
  loses bytes on your setup.

  After a successful connection, the package utilities can use the current
  network settings until DSS is restarted.

Exit status
  0  Network bring-up completed.
  2  Sprinter-WiFi hardware was not found.
  3  ESP communication error.
  4  NET.CFG is missing, unreadable or incomplete.
