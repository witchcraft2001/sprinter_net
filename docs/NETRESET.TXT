NETRESET.EXE - Reset and reinitialize ESP

Purpose
  Performs a hardware reset of the SprinterESP / Sprinter-WiFi ESP module and
  applies basic ESP defaults used by the network kit.

Usage
  NETRESET.EXE

When to use
  Use NETRESET when the ESP-AT stream is confused, after manual WTERM testing,
  or before repeating diagnostics after a failed network command.

Typical recovery sequence
  NETRESET.EXE
  NETPROBE.EXE
  NETUP.EXE
  TCPTEST.EXE

Exit status
  0  Reset and initialization completed.
  2  Sprinter-WiFi hardware was not found.
  3  ESP communication error.

Notes
  In MAME with jesperl, some emulator-side TCP operations may block the emulator
  process. In that case NETRESET may not get an answer until jesperl finishes
  the pending host-side operation.

  NETRESET is a recovery tool and uses the default ESP startup UART speed
  115200 after reset. Run NETUP again to apply BAUD from NET.CFG.
